Présentation

Nous avons décider de dévier un peu de la présentation proposer dans l'énoncé du travail pour permettre une expérience utilisateur un peu plus complète. L'utilisateur n'a pas besoin de recharger la page pour recommencer une nouvelle partie et il peut changer la taille de la grille lui-même grâce aux boutons de l'interface.

Pour ce qui est de la police de caractère, nous avons opté pour une police trouvé sur Google Fonts du nom de Quicksand.

Le jeu est aussi jouable sur tous les grandeurs d'écran.

Apprentissages

Durant ce travail, nous avons compris plusieurs concepts importants du Web.

D'abord, pour ce qui est du style, nous avons dû travailler fort pour faire en sorte que le jeu soit bien présenté peu importe la grandeur de l'écran. Nous avons tenté de contenir cette tâche uniquement dans le CSS, mais nous avons parfois été obligé d'utiliser du Javascript pour ajuster le style.

Par la suite, nous avons dû gérer les entrées claviers de l'utilisateur et cela nous a causé quelques problèmes, car pour un seul clique de touche, plusieurs évènements étaient déclanché, ce qui donnait un comportement innatendu au jeu. Pour régler ce problème, il a fallu ajouter un appel à la fonction off de jQuery, qui unbind l'évènement après son premier déclenchement.

Finalement, nous avons aussi voulu rendre le jeu le plus personalisable possible. Il est donc facile de changer plusieurs attributs du jeu, soit la fréquence d'apparition d'un 2 (défaut : 90%), la valeur minimale d'une tuile (défaut : 2), la valeur maximale (défaut : 65536) et la valeur de victoire (défaut : 1024 * valeur minimale). Tous ces attributs sont facilement ajustables dans le Javascript avec des variables globales.
